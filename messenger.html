Overview

With the rise of front-end frameworks and libraries, I wanted to try out React
, a JavaScript library for building UI, and learn more about front-end engineering. Along with React, I also decided to learn Redux
, which is a state management JavaScript library.

Along with learning these new libraries, I also wanted to develop an interesting app with technical and design challenges. Therefore, I decided to build the Facebook's Messenger
 app for desktop using Electron, because I believed that building the Messenger from scratch could be a great way to improve my development skills, as well as my ability to analyze a product.

Despite Facebook's extensive developer resources
, there was no way to directly interact with user account functionalities, i.e. messaging a friend. Therefore, I decided to utilize Facebook-Chat-API
, which is an unofficial API for interacting with Facebook user account.

To summarize what I hope to accomplish:
  • Learn more about front-end engineering through React
  • Write clean, modular code
  • Analyze a product such as Messenger
  • Come up with new features, possibly specific to desktop
  • ...

Login screen.

LOGIN

The login page was simple. I referred to the basic functionality on messenger.com
 and added custom styling. As for "Remember me", I utilized electron-settings
 to replace cookie's functionality and temporarily save login instance and user's name.

While maintaining the functionalities, I wanted add my own improvements to the input boxes. Therefore, I referred Android version of Messenger (which was actually adhering to Google's Material-Design
) to build a different UI that fitted the theme better.


Loading animation.

Another aspect that I wanted was adding a drop shadow. To add a custom drop shadow, I tinkered with Electron and essentially made the app window transparent (full control with CSS).

I was faced with several minor technical issues when I wanted to implement a smooth transition in between routing (rather than glitching). To tackle the issue, I decided to use Electron's functions to show/hide based on certain event dispatches (such as when login process is complete).

Another solution that I would love to try, possibly within the app, is to learn and use react-transition-group
.


Application screen.

Application

There were a lot of decisions to make: the most basic one being how to divide up the application in terms of React components and a difficult one being what to store for each component's state. Unlike my previous experiences with traditional JavaScript with jQuery, working with React required a lot more planning and reasons to back up every decisions.

I decided to divide the main application page into three main components: ThreadList, ThreadWindow, and ThreadDetail. Because all three depend on current thread that user is viewing (especially window and detail), I decided to connect them by defining a "currThread" component-level state on the parent component (App).

Without a doubt, there are many ways to go about this particular problem. I could have enabled Redux's application-level state on the three components, making them a "smarter" component. Another way could've been along the lines of utilizing the Electron/Node.js back-end.

Each component handled a lot of complex functions (in fact, there is still more to add) through Chat API, React's component functions, etc. Some of the interesting ones that deal with parsing data include: having to convert a hex to rgb values so the app updates its theme color, parsing timestamps in an unique pattern relative to current time, etc.


Resizing textbox.

Another small technical challenge that I enjoyed was implementing a resizing textbox. Because if a user types beyond an input box, the text either overflows or extends and the box does not necessarily responsively resize.

To solve this challenge, I looked into the textbox DOM property of scrollHeight since the text overflowed whenever it exceeded the box size. With that information, I was able to easily resize the box. I was also able to add another working functionality with "shift+enter", which adds a new line at cursor, on top of resizing function.


Takeaway

The project is still a work-in-progress. The side project turned out be a lot more complex than I expected it to be. Because I had already learned so much, I went ahead and wrote this analysis to reflect on my progress and gains through this project.

Because there is still much more to learn about React, as well as Redux, I will be focusing on what I want to implement next:
  1. Animations and/or transitions. I am a fan of smooth transitions, but React's documentations show that they do not have support for transitions. A lead that I found is react-transition-group
    .
  2. Code structure. Since React is a library, I have full control on how to structure this project. Mostly, I've been focused on class-based components. However, I definitely want to take advantage of function-based components and learn to improve overall organization.
  3. New features. Because I am building this application from scratch, I am able to add or adjust any aspects to whatever I want. A goal I have is to add a new look/design specific to desktop for this app in the near future.
