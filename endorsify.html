Overview

During summer 2017, I had a chance to work as a software engineer intern for a marketing startup, Endorsify
, in Los Angeles. Endorsify is a SaaS (Software as a Service) platform that helps marketers establish connections with content creators.

I mostly worked on the web application's front-end, or the "view" of Laravel's MVC architecture. Throughout the internship, I was able to contribute to several projects by writing production code that got shipped onto the live web app.

Here is what I did during the internship:
  • New pricing page
  • Strategy calculator (CPM, CPE)
  • New landing page
  • Data Visualization Dashboard
  • LOTS of edits and bug fixes (i.e. ticket requests)


Pricing

As I began the internship, I was assigned a simple project of updating the pricing page in order to kickoff my internship and familiarize myself with the development environment (PHP Laravel, Git), as well as setting up a new environment (.env configuration, dependencies, api keys, etc.).

New pricing page.



After receiving the design for new pricing page from our lead designer, I built this page by modifying modal and button components from Materialize
, a CSS framework inspired by Material Design, and other assets from the project.

I was also given a style guide to adhere to throughout the internship. As I built UI components, I had to make sure its font style, font size, color, margin, and other relevant properties followed the criterias listed on the style guide.


Calculator

Strategy calculator is a feature built for marketers to use. It takes five inputs to calculate CPM and CPE (Cost per Impression and Cost per Engagement).

Strategy calculator.


I built the calculator using modal and input components from our CSS framework and animations from SweetAlert2
 while adhering to given design. I added animations (modal closing, output)
, which you can see here
,
to the feature. The calculator checks for validation with regex (throws an error alert).

On the back-end, the calculator asks for confirmation and saves the user's inputs onto database so that the data science team can utilize these values. Users are also able to revisit these inputs.


Landing

Building the new landing page was probably one of the most important contributions I made during the internship. I had to finish the new landing page under deadline before Endorsify's alpha launch.


New landing page.

It was important for me to develop the page as precise to the design. Also, I had to make sure the new page was mobile-responsive (mobile look
). I created an animation on the waves below the hero illustration (you can see it on Endorsify
). I utilized SVG's scalability and CSS's overflow property.

Developing this page was not entirely new to me. So I was able to take up this chance to build a more structured page of organized HTML and CSS by taking advantage of different HTML tags, as well as cleaner CSS classes.


Dashboard

The data visualization dashboard was to be used by the marketers and give access to data in an interactive way.


Data visualization dashboard.

I used D3
 to develop the dashboard. D3 is a JavaScript library that allows data visualization using HTML, SVG, and CSS. D3 allows users to have a lot of control over what they build. As powerful D3 is, there was a lot for me to learn to use D3.

The dashboard had 4 charts (3 types: world map, bar chart, bubble chart). While D3 provided many excellent examples
, there was a lot to be done. For example, to build a world map, I had to work with JSON data (TopoJSON
) and decide how to project the map (mercator, equirectangular, etc.).


Bubble chart.

I also had to append a modal of data to each element (bar, bubble, country, etc.) that appeared on mouse hover.

When I was working on the bubble chart, I decided that I wanted to add cool animations to the chart. To do so, I referred to D3's force
, which I had to spend time a lot of time learning how to tinker with its properties and D3's physics implementation. D3's force and gravity property enabled for me to build really cool animations
, which you can see here
,
on the bubble chart.

Takeaway

Working at Endorsify was a learning experience. I produced tangible results and made contributions to the team.

Here is what I've learned during the internship:
  1. Building real ideas. I had direct exposure to a large codebase, where I was able to write production code.
  2. Utilizing resources. There are a lot of libraries and frameworks that already solved most problems. These resources helped my development by providing me with reference to building out my ideas.

And here is where I could've done better:
  1. Better approach. There were many instances where I felt inefficient. For example, instead of using D3, I could have used Chart.js
    , which is a much simpler framework for data visualization on web.
  2. Varied work. Although I mostly worked on the front-end, I was open to working on the back-end as well as long as I was willing/able to. It would have been cool if I had worked and contributed more on the back-end.
